Padding Oracle Attack Simulation

This project focused on simulating a Padding Oracle Attack to exploit vulnerabilities in CBC-mode encryption. The attack demonstrated how insecure padding in cryptographic protocols could be leveraged to decrypt sensitive data without access to the encryption key.

Key Features and Functionality:

  • Ciphertext Decryption: Successfully decrypted encrypted messages by manipulating ciphertext blocks and analyzing server responses.
  • Risk Demonstration: Highlighted the dangers of insecure padding mechanisms in real-world encryption protocols.
  • Python Implementation: Scripts automated the attack process, demonstrating the practical feasibility of such exploits.

Development Process:

  • Python: Utilized for scripting the attack logic, handling encryption and decryption processes, and simulating server responses.
  • Cryptography Concepts: Applied cryptographic principles like CBC-mode encryption and padding schemes to identify vulnerabilities.

Implementation:

  • Crafted malicious ciphertexts and sent them to a simulated server to observe oracle responses.
  • Iteratively adjusted ciphertexts based on server feedback to reconstruct the original plaintext.
  • Documented the process to provide a clear understanding of how padding oracle attacks operate and their implications.

Impact and Outcome:

The simulation highlighted the importance of adopting secure cryptographic practices, such as avoiding predictable padding schemes or implementing robust defenses like authenticated encryption. It provided valuable insights for developers and security professionals, demonstrating the critical need for secure encryption design. The project reinforced the significance of using modern encryption protocols like AES-GCM to mitigate such vulnerabilities.

Kaminsky DNS Cache Poisoning

This project focused on the Kaminsky DNS Cache Poisoning Attack, which exploits vulnerabilities in the DNS protocol to manipulate cached records. By demonstrating how malicious entities can redirect users to fraudulent websites, the project emphasized the need for securing DNS infrastructures through modern solutions like DNSSEC.

Key Features and Functionality:

  • DNS Spoofing Demonstration: Successfully poisoned DNS caches to redirect legitimate traffic to malicious sites.
  • Protocol Analysis: Highlighted inherent flaws in DNS, such as the lack of authentication in traditional systems.
  • Security Advocacy: Showcased DNSSEC as a robust solution for mitigating cache poisoning attacks.

Development Process:

  • DNS Configuration Tools: Configured DNS servers to set up the attack environment.
  • Networking Knowledge: Applied concepts like TTL (Time-to-Live), resolver behavior, and cache overwriting techniques.

Implementation:

  • Deployed a controlled DNS server environment to simulate the attack without real-world harm.
  • Crafted spoofed DNS responses to overwrite legitimate entries in the cache.
  • Analyzed traffic redirection and documented the risks associated with insecure DNS configurations.

Impact and Outcome:

This project demonstrated how attackers could exploit the lack of validation in DNS responses, leading to severe consequences like phishing attacks, data theft, and service disruptions. By highlighting the vulnerabilities in traditional DNS, it reinforced the need for widespread adoption of DNSSEC to ensure data authenticity and integrity. The project served as an educational tool for security professionals, enabling better defense strategies against DNS attacks.
